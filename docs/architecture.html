Request / Data Flow

01 · USER

Filter interaction

User changes a filter in FilterBar.vue. The useFilters singleton updates a shared reactive ref (period / warehouse / category / status).

02 · COMPOSABLE

useFilters

getCurrentFilters() maps UI state to query params. selectedPeriod becomes month=YYYY-MM or quarter e.g. Q3-2025.

03 · API CLIENT

api.js (Axios)

The active view calls an api.* method. Axios issues GET /api/…?warehouse=…&category=… to localhost:8001.

04 · BACKEND

FastAPI route

apply_filters() walks the in-memory list for warehouse, category & status. filter_by_month() handles exact months and quarter ranges (Q1–Q4).

05 · VALIDATION

Pydantic models

FastAPI serialises the filtered list through a response_model. Fields are coerced and validated; extra fields are stripped. Returns JSON array.

06 · REACTIVITY

Vue computed props

Raw data lands in a ref (e.g. allOrders). Derived values (totals, counts) live in computed() and re-render the template automatically.

Global Filter System

📅
Time Period
all · YYYY-MM · Q1–Q4 2025
→ month param
🏭
Warehouse
all · specific location
→ warehouse param
🏷️
Category
all · product category
→ category param
📋
Order Status
all · Processing · Delivered · Backordered · ...
→ status param
Implemented as a module-level singleton in useFilters.js using plain ref() variables declared outside the composable function. This means all views and components share the same reactive filter state without Vuex/Pinia. The FilterBar component drives state; each view calls getCurrentFilters() and re-fetches on every filter change. Note: Inventory does not support the month filter (no time dimension on stock levels).
